## Learned Workspace Facts

- Search Merge Requests: UI in `src/components/mr_search.tsx` (entry `src/mr_search.tsx`); AI tool `src/tools/search-merge-requests.ts` is separate. `MyProjectsDropdown` with `includeAllItem={false}` (project required before load); sections "Created by me", "Assigned to Review" (hidden when empty), "Other"; Sort By (`mr_sort.tsx`, ⌘S) sets GitLab `order_by` ("Default" omits param).
- `MyProjectsDropdown` in `src/components/project.tsx` supports `includeAllItem` (default `true`); My Merge Requests and Reviews keep "All Projects".
- My Merge Requests hides author subtitle and avatar on list rows via `showAuthor={false}` on `MRListItem`; Search MR and other MR lists keep the default (`showAuthor` true). MR list row subtitle is hidden while the side panel is open.
- MR list side panel: `useMRListDetails()` + `MRListDetailsToggleAction` (⌘⇧D). Metadata column: `useMRListMetadata()` + `MRListMetadataToggleAction` (⌘⇧I, only when side panel open). Side panel uses `MRListDetailMetadata`, full detail uses `MRDetailMetadata` in `mr_metadata.tsx`; the former `listdetails` preference was removed from `package.json`.
- Shared MR UI in `mr.tsx`, `mr_actions.tsx`, `mr_metadata.tsx`, `mr_discussions.ts`, `mr_status.ts`. Do not fetch MR discussions per list row; `useMRDiscussionStats` in `mr_discussions.ts` is for detail/side panel only. Metadata label is "Discussions" (`resolved/resolvable` only; not `user_notes_count`); no discussion count in list accessories.
- My Merge Requests and Reviews use GitLab API default order (`created_at` desc); no client-side re-sort.
- GitLab [Merge requests API](https://docs.gitlab.com/api/merge_requests/): list endpoints may return stale `merge_status` / `has_conflicts` unless `with_merge_status_recheck=true` (async recheck, not guaranteed; may be ignored on 15.11+ without Developer+ if `restrict_merge_status_recheck` is enabled). `with_merge_status_recheck` does not control `head_pipeline`. Avoid `view=simple` when full MR fields are needed. Extension defaults in `getMergeRequests` / `getGroupMergeRequests`: `with_labels_details=true` and `with_merge_status_recheck=true` (callers can override).
- `MRListItem` must not call GitLab per row for approvals, full MR, or discussions—only `useMRPipelines(mr, { limit: 1 })` when list payload has no `head_pipeline` (and not when `has_conflicts`). Pipeline accessory: prefer `head_pipeline` from list (`getMRHeadPipelineStatus`, `parseHeadPipelineFromJson`); else `useMRPipelines(mr, { limit: 1 })`; `getCIJobStatusIcon`; hidden when `has_conflicts`. Full MR pipelines use `useMRPipelines(mr, { all: true })` in `MRPipelineList` (separate cache key). Discussions/full MR/approvals fetches belong in side panel (`MRListDetail`) or detail (`MRDetail`) only.
- Pipeline list (`pipelines.tsx`): subtitle short SHA; date accessory `formatPipelineListAccessoryDate` in `utils.ts`. Pipeline row actions (`pipeline_actions.tsx`): Retry (confirmAlert, ⌘R), Cancel (destructive), Run Pipeline (confirmAlert, ⌘N), Copy URL (⌘C) in first section; Refresh without ⌘R. Jobs list (`jobs.tsx`): no accessories; status on icon tooltip; `manual` → `Icon.Gear` in `getCIJobStatusIcon`. Job actions (`job_actions.tsx`): Retry/Cancel/Run, Copy URL (⌘C), Download Artifact submenu (⌘D, only when `artifacts` non-empty); Refresh without ⌘R. Artifact download: preference `artifactDownloadDirectory` (default `~/Downloads`); `trace` → `/jobs/:id/trace`, `archive` → `/jobs/:id/artifacts`; open with `open()` not `showInFinder`. Prefer REST jobs fetch (`pipelineID`) for `artifacts` field.
- Commits list (`commits/`): no per-row `useCommitStatus` in `CommitListItem` or `EventCommitListItem`; static commit icon. `useCommitStatus` (`commits/utils.ts`) only for `branch.tsx`. `MRCommitList` uses MR commits API; `ProjectCommitList` uses repository commits with pagination (`commits/data.ts`).
- Author subtitle/tooltip email uses `User.public_email` when the API provides it. Import the `gitlab` client from `src/common.ts`, not `src/gitlabapi.ts`.
- MR list data migration (in progress): move `useMyMergeRequests` (`mr_my.tsx`) and Search MR fetch (`mr_search.tsx`) from file `useCache` to `useCachedPromise`; Search MR UI prefs already use `useCachedState`. Remove "Clear Local Extension Cache" (`cache_actions.tsx`); keep file `useCache` for projects, pipelines, issues, etc. `useMyMergeRequests` cache args exclude project—project filter stays client-side in `MyMergeRequests`.
